# UX仕様

## 1. 画面構成

単一ページ構成とし、クライアントサイドルーティングは使わない。

デスクトップでは三列構成とする。

左列はピーストレイとパズル操作、中央列は盤面、右列は資源、アップグレード、Auto Solver状態を表示する。

横幅が狭い場合は、盤面、ピーストレイ、操作、ソルバ、アップグレードの順に一列へ積む。盤面は画面幅を超えない。セルサイズは最小28px、最大64pxを目安に可変とする。

## 2. ヘッダー

ヘッダーには以下を表示する。

- 仮タイトル
- 現在Compute
- 直近の毎秒探索ノード数
- Settingsボタン
- Statsボタン

Computeは整数表示を基本とし、1,000以上は桁区切りを付ける。1,000,000以上では短縮表記を併記してよいが、ツールチップまたは詳細画面で正確な値を確認可能にする。

## 3. 盤面

使用可能セル、ブロックセル、空セル、配置済みセルを視覚的に区別する。

選択中ピースを置ける位置へマウスまたはタッチを移動した際、仮配置を半透明で表示する。合法なら通常のプレビュー、不正ならエラー表示にする。

Auto Solver動作中は探索中の仮配置を別レイヤーで描画する。手動配置と混同させない。探索が高速な場合は更新を間引く。

盤面上部に以下を表示する。

- Tier
- seed
- 難易度スコア
- 経過時間
- 現在のクリア区分見込み

seedにはコピーボタンを付ける。URL共有はMVPでは不要である。

## 4. ピーストレイ

未配置ピースを個体ごとに表示する。同型ピースが複数ある場合も別カードにする。

クリックまたはタップで選択する。ドラッグ開始でも選択できる。

選択中ピースは明確な枠で示す。回転後の形状をトレイ上にも反映する。

配置済みピースはトレイ上で薄くするか、別の`Placed`領域へ移す。個体識別子は内部に保持するが、通常UIでは `T #2` 程度の短い表示でよい。

## 5. 入力

必須操作は以下とする。

| 入力 | 動作 |
|---|---|
| 左クリック／タップ | ピース選択、配置 |
| ドラッグ | ピース移動 |
| 右クリックまたは `R` | 時計回り回転 |
| `Shift+R` | 反時計回り回転 |
| `Q` / `E` | 反時計回り／時計回り回転 |
| `Z` または `Ctrl+Z` | Undo |
| `Y` または `Ctrl+Shift+Z` | Redo |
| `Escape` | 選択解除 |
| `Delete` / `Backspace` | 選択中の配置済みピースを取り外す |
| `H` | 解禁済みならPlacement Scanner切替 |
| `Space` | Auto Solver開始／一時停止。入力欄フォーカス時は無効 |

右クリック時にブラウザのコンテキストメニューを抑止するのは、盤面とピース上だけとする。

タッチ環境では、ピースカード内に回転ボタンを表示する。ドラッグだけに依存してはならない。

## 6. 操作ボタン

盤面周辺に以下を置く。

- New Puzzle
- Reset Board
- Undo
- Redo
- Rotate Left
- Rotate Right
- Remove Piece
- Hint
- Forced Move
- Start/Pause Solver

未解禁または現在使用不能な機能はdisabledにする。disabledの理由をツールチップか短い説明で表示する。

`New Puzzle` は未クリア盤面を破棄するため、配置が一つ以上ある場合だけ確認を出す。

## 7. アップグレード画面

右列またはモーダル内にカード形式で表示する。

各カードには名称、効果、現在レベル、次レベル効果、価格、購入ボタンを含める。

購入可能、資金不足、最大レベル、前提未達を視覚的に区別する。

購入後は即時保存する。

## 8. Auto Solverパネル

以下を表示する。

- 状態: Idle / Running / Paused / Solved / Failed
- 対象Tierとseed
- 探索ノード数
- バックトラック数
- 実測nodes/s
- 理論nodes/s
- 探索深さ
- キュー長
- 論理並列セッション数
- Start、Pause、Cancel

Cancelはそのセッションの探索状態を破棄する。報酬は発生しない。

`Failed` は解なし、内部上限到達、生成不整合を区別する。解なしが生成器由来なら、バグとしてコンソールへ詳細を出し、同seedを保持する。

## 9. クリア表示

クリア時には短い結果モーダルを表示する。

- Clear分類
- 獲得Compute
- 難易度スコア
- 手動時間または探索時間
- 探索ノードとバックトラック
- Next Puzzle
- Close

Auto Queueが有効な場合、モーダルで処理を止めない。右下の非モーダルトーストへ縮退させ、次の問題を開始する。

## 10. 設定

以下を実装する。

- 探索アニメーション: On / Reduced / Off
- アニメーション速度
- 効果音: On / Off。MVPで音源未実装なら設定自体を省略してよい
- 高コントラスト表示
- セーブExport
- セーブImport
- Erase Save
- バージョン表示

OSの`prefers-reduced-motion`が有効なら探索アニメーションを初期状態でReducedにする。

## 11. アクセシビリティ

色だけで合法／不正／選択を区別しない。枠線、パターン、アイコンのいずれかを併用する。

すべてのボタンにキーボードフォーカスを表示する。

盤面セルには過剰な個別tab stopを設けない。キーボード操作は選択ピースの仮アンカー座標を矢印キーで移動し、Enterで配置する方式を採用してよい。

主要な状態変化は`aria-live="polite"`へ短く通知する。高速探索イベントは読み上げない。

## 12. エラー表示

予期可能な操作エラーはトーストまたは盤面内表示にする。スタックトレースをUIへ出さない。

保存失敗、Worker停止、セーブ移行失敗など継続性に関わるエラーは、画面上部へ消えない警告として表示する。
